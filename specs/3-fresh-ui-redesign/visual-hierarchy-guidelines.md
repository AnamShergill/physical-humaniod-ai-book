# Visual Hierarchy and Depth Guidelines

## Overview
This document outlines the visual hierarchy and depth system implemented in the fresh UI redesign. It provides guidelines for maintaining consistent visual design across all pages.

## Color Hierarchy
- **Primary Color**: Used for main headings, important buttons, and primary actions
- **Secondary Color**: Used for secondary elements, links, and supporting information
- **Accent Color**: Used for highlights, notifications, and special emphasis
- **Emphasis Colors**: Used for different levels of text importance

## Depth System
- **Level 0**: Base background (no shadow)
- **Level 1**: Light content containers (light shadow)
- **Level 2**: Main content blocks (medium shadow)
- **Level 3**: Important cards and featured content (heavy shadow)
- **Level 4**: Floating elements and modals (extra heavy shadow)
- **Level 5**: Top-level elements (ultra heavy shadow)

## Typography Hierarchy
- **H1**: 2.5rem - Page titles and main headings
- **H2**: 2rem - Section titles
- **H3**: 1.5rem - Subsection titles
- **H4**: 1.25rem - Minor headings
- **H5**: 1.1rem - Small headings
- **H6**: 1rem - Label headings
- **Body**: 1rem - Main content text
- **Small**: 0.875rem - Supporting text

## Spacing Hierarchy
- **Small**: 0.5rem - Tight spacing for related elements
- **Medium**: 1rem - Standard spacing between elements
- **Large**: 2rem - Major section separations
- **X-Large**: 3rem - Important visual breaks

## Component Hierarchy
- **Cards**: Use depth levels to indicate importance
- **Buttons**: Primary, secondary, and accent styles
- **Navigation**: Clear visual indicators for active states
- **Content Blocks**: Different styles for different content types

## Responsive Considerations
- Depth effects are reduced on smaller screens for performance
- Typography scales appropriately across devices
- Interactive elements maintain adequate touch targets
- Layout adapts while preserving visual hierarchy

## Accessibility Guidelines
- All color combinations meet WCAG 2.1 AA contrast requirements
- Interactive elements have clear focus states
- Depth effects don't compromise readability
- Text remains legible across all background colors
